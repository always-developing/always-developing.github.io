Downcasting automatically with dynamic keyword

How the dynamic keyword can be used to automatically downcast a variable

Mar 21, 2022
5 minute read

Daily Knowledge Drop

The dynamic keyword can be used to to automatically downcast a variable to the correct type when calling a method.

This technique is useful when having a variable of a base type, and a method needs to be called using a derived type.

Lets look at a few examples, which will help make things a bit clearer.

Base setup

In the examples below, the following hierarchy of classes is used:

public class Vehicle
{
    public void PrintVehicle() => 
        Console.WriteLine($"I am a {typeof(Vehicle).Name}");
}

public class LandVehicle : Vehicle
{
    public void PrintLandVehicle() => 
        Console.WriteLine($"I am a {typeof(LandVehicle).Name}");
}

public class Car : LandVehicle
{
    public void PrintCar() => 
        Console.WriteLine($"I am a {typeof(Car).Name}");
}

public class SeaVehicle : Vehicle
{
    public void PrintSeaVehicle() => 
        Console.WriteLine($"I am a {typeof(SeaVehicle).Name}");
}

public class Boat : SeaVehicle
{
    public void PrintBoat() => 
        Console.WriteLine($"I am a {typeof(Boat).Name}");
}

The hierarchy looks as follows. Each level has a print method which prints out which type it is:

  • Vehicle:
    • Land Vehicle
      • Car
    • Sea Vehicle
      • Boat

We also have a VehicleHelper class, which will assist in dealing with Vehicles:

public class VehicleHelper
{
    public void PrintVehicle(Car c)
    {
        c.PrintCar();
    }

    public void PrintVehicle(Boat b)
    {
        b.PrintBoat();
    }
}

This class can print a vehicle, but only if its a Car or a Boat.

Working example

Using the VehicleHelper, we can print out a Car or Boat as follows:

Boat boat = new Boat();

Console.WriteLine("=====");
helper.PrintVehicle(boat);

The problem

Assume the VehicleHelper class was extended to include a factory method to return a Vehicle. We can ask it for a Vehicle of a particular type by using a string, and it will return that type.

public class VehicleHelper
{
    public void PrintVehicle(Car c)
    {
        c.PrintCar();
    }

    public void PrintVehicle(Boat b)
    {
        b.PrintBoat();
    }

    public Vehicle GetVehicle(string type) => type switch
    {
        "vehicle" => new Vehicle(),
        "land" => new LandVehicle(),
        "car" => new Car(),
        "sea" => new SeaVehicle(),
        "boat" => new Boat(),
        _ => new Vehicle()
    };
}

This method can now be used to get new Vehicles:

1
2

Vehicle vehCar = helper.GetVehicle("car");
Vehicle vehBoat = helper.GetVehicle("boat");

The problem here, is that even though we are asking for a Car and the method instantiates and returns a Car, the return type is Vehicle - the Car (or boat, or any other type) gets upcast to type Vehicle.

This means the following will NOT compile:

1
2
3
4
5

var helper = new VehicleHelper();

Vehicle vehCar = helper.GetVehicle("car");

helper.PrintVehicle(vehCar); // ERROR on this line

The error on line 5 is cannot convert from 'Vehicle' to 'Car'.

Even though vehCar was instantiated as a Car is was upcast to a Vehicle and there is no PrintVehicle method which accepts type Vehicle.

The fix

The dynamic keyword can be used to automatically downcast the Vehicle to the correct type, and have the correct method invoked.

1
2
3
4
5
6
7
8

var helper = new VehicleHelper();

Vehicle vehCar = helper.GetVehicle("car");
Vehicle vehBoat = helper.GetVehicle("boat");

helper.PrintVehicle((dynamic)vehCar);
Console.WriteLine("=====");
helper.PrintVehicle((dynamic)vehBoat);

On lines 6 and 7, the Vehicle is cast to dynamic - the result is that the vehCar will be cast to Car, and the correct PrintVehicle method called, with the same pattern applying to vehBoat.

The output is as follows:

1
2
3

I am a Car
=====
I am a Boat

This is incredible useful - just by casting it to dynamic, the compiled will cast to the correct type and invoke the correct method!

Notes

dynamic has a bit of a negative reputation, but in this case it proves to be very useful. As always though, especially if this is used in a critical hot path, the performance should be benchmarked against other methods to determine if using dynamic makes sense. It might make more sense, for example, to manually check and cast the type (using the is keyword) and calling the appropriate method.
